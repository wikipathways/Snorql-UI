```sparql
# title: Count of {{entityType}}
# category: Metadata
# description: Counts the total number of a selected entity type in WikiPathways.
# param: entityType|enum:DataNode=Data Nodes,GeneProduct=Gene Products,Interaction=Interactions,Metabolite=Metabolites,Protein=Proteins|DataNode|Entity Type
```

**Create averagePerPathway.rq** (TMPL-02, per D-01 separate template):

```sparql
# title: Average {{entityType}} per Pathway
# category: Metadata
# description: Calculates the average, minimum, and maximum number of a selected
#   entity type per pathway in WikiPathways.
# param: entityType|enum:DataNode=Data Nodes,GeneProduct=Gene Products,Interaction=Interactions,Metabolite=Metabolites,Protein=Proteins|DataNode|Entity Type

SELECT (AVG(?no) AS ?avg)
       (MIN(?no) AS ?min)
       (MAX(?no) AS ?max)
WHERE {
    SELECT ?Pathway (COUNT(?entity) AS ?no)
    WHERE {
    ?entity a wp:{{entityType}} ;
       dcterms:isPartOf ?Pathway .
    ?Pathway a wp:Pathway .}
}
```

**Create pathwaysForDatasource.rq** (TMPL-03, per D-06 enum, per D-15 `wp:bdb{{datasource}}`):

```sparql
# title: WikiPathways for {{datasource}} Identifiers
# category: Data Sources
# description: Lists pathways containing entities with identifiers from a selected
#   data source, showing the pathway title and identifier URI.
# param: datasource|enum:Chemspider=ChemSpider,Ensembl=Ensembl,HgncSymbol=HGNC,Hmdb=HMDB,EntrezGene=NCBI Gene,PubChem=PubChem|Ensembl|Data Source

SELECT DISTINCT ?pathwayRes (str(?wpid) AS ?pathway) (str(?title) AS ?pathwayTitle) ?identifier
WHERE {
  ?gene dcterms:identifier ?id ;
    dcterms:isPartOf ?pathwayRes ;
    wp:bdb{{datasource}} ?identifier .
  ?pathwayRes a wp:Pathway ;
    dcterms:identifier ?wpid ;
    dc:title ?title .
}
```

CRITICAL: Enum values must be exact WikiPathways predicate suffixes (case-sensitive): `Chemspider` (not `ChemSpider`), `Ensembl`, `HgncSymbol` (not `HGNC`), `Hmdb` (not `HMDB`), `EntrezGene` (not `NCBI`), `PubChem`. The display labels use human-friendly names via value=label syntax. The `fn:substring` extraction from original queries is intentionally dropped because the offset varies per datasource and cannot be parameterized; instead the full identifier URI is returned.

**Create countEntityPerSpecies.rq** (TMPL-04, per D-16 Mustache in type):

```sparql
# title: {{entityType}} per Species
# category: Metadata
# description: Counts the number of distinct entities of a selected type per species
#   in WikiPathways, ordered by count descending.
# param: entityType|enum:DataNode=Data Nodes,GeneProduct=Gene Products,Metabolite=Metabolites,Protein=Proteins|DataNode|Entity Type

SELECT (COUNT(DISTINCT ?entity) AS ?count) (str(?label) AS ?species)
WHERE {
  ?entity a wp:{{entityType}} ;
    dcterms:isPartOf ?pw .
  ?pw dc:title ?title ;
    wp:organism ?organism ;
    wp:organismName ?label .
} GROUP BY ?label ORDER BY DESC(?count)
```

Note: This enum has 4 values (no Interaction) because no `countInteractionsPerSpecies.rq` existed in the original set. Per research analysis.
