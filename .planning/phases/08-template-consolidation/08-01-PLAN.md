**snorql.js changes (3 modifications):**

1. **parseRqHeaders enum parsing** (~line 152-154): Replace the current `paramOptions` assignment with value=label object parsing. Change:
```javascript
paramOptions = paramType.substring(5).split(',').map(function(o) { return o.trim(); });
```
To:
```javascript
var rawOptions = paramType.substring(5).split(',').map(function(o) { return o.trim(); });
paramOptions = rawOptions.map(function(o) {
    var eqIdx = o.indexOf('=');
    if (eqIdx > 0) {
        return { value: o.substring(0, eqIdx), label: o.substring(eqIdx + 1) };
    }
    return { value: o, label: o };
});
```
This implements D-10 (enum value=label syntax). When `=` is present, left side is substitution value, right side is display label. When absent, both are the same (backward compatible).

2. **sanitizeEnumValue** (~line 196-201): Replace the function body to check `.value` property on option objects:
```javascript
function sanitizeEnumValue(value, allowedOptions) {
    if (!allowedOptions) return null;
    for (var i = 0; i < allowedOptions.length; i++) {
        if (allowedOptions[i].value === value) return value;
    }
    return null;
}
```

3. **buildParamPanel enum rendering** (~line 469-475): Update the `<option>` generation loop to use `.value` and `.label` properties:
```javascript
if (p.type === 'enum' && p.options) {
    html += '<select class="form-control param-input" id="param-' + p.name + '" data-param="' + p.name + '">';
    for (var j = 0; j < p.options.length; j++) {
        var opt = p.options[j];
        var selected = (opt.value === p.defaultValue) ? ' selected' : '';
        html += '<option value="' + opt.value + '"' + selected + '>' + escapeHtml(opt.label) + '</option>';
    }
    html += '</select>';
}
```
Note: Use `escapeHtml(opt.label)` for the display text to prevent XSS from label content. The `opt.value` in the value attribute is safe because it passes through sanitizeEnumValue before SPARQL substitution.
